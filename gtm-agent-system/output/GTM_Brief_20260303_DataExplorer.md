## 3. Before/After

### 업무 변화 대비표

| 업무 단계 | Before (현재) | After (Phase 1) |
|----------|--------------|----------------|
| 속성 탐색 | Braze에서 유사 이름 속성을 하나씩 클릭해 추측 (10~15분) | 통합 카탈로그에서 이름·설명·예시값을 한 화면에서 확인 (2~3분) |
| 명세서 확인 | Confluence 문서 검색 → 최신 버전 여부 불명확 (20~30분) | 메타데이터 패널에서 설명·수집 상태를 즉시 조회 (0분 대기) |

| **합계 리드타임** | **평균 4시간 (대기 포함)** | **30분 이내** |

## 4. Key Message

### Primary Message

**원하는 데이터를 기다림 없이 스스로 찾는다**

### Supporting Messages

1. **효율 개선**: 새 속성 하나를 캠페인에 쓰기까지 4시간 걸리던 과정이 30분으로 줄어든다. Slack 문의 없이, 대기 없이.
2. **정확성·신뢰성**: 설명·예시값·최근 7일 수집 현황이 한 화면에 표시되어, 추측이 아닌 검증된 데이터로 세그먼트를 만든다.
3. **협업·가시성**: 어느 오디언스·캠페인에서 이 속성이 쓰이는지 활용 맥락이 연결되어, 팀 전체가 같은 정보를 보고 의사결정한다.

### Phase 1 범위 안내

> Phase 1에서는 데이터 탐색·조회 환경을 제공합니다. 속성 수집 On/Off 제어, 자연어 검색(LLM 기반)은 Phase 2·3에서 지원될 예정입니다.

## 5. What's in/out

### Phase 1에서 제공 (What's in)

- **통합 카탈로그 뷰**: 유저 행동(이벤트·이벤트속성·커스텀이벤트), 유저 프로필(일반·추론 속성), 유저 통계 데이터를 하나의 탐색 화면에서 조회
- **메타데이터 패널**: 속성 설명, 데이터 타입, 예시값, 수집 상태(Active/Warning/Inactive), 최근 7일 일별 수집 추이를 사이드 패널에서 즉시 확인
- **활용 맥락(연결 현황)**: 속성이 연결된 오디언스·개인화 변수·캠페인 정보를 패널에서 조회
- **속성 정보 자동 동기화**: 매일 1회 실제 수집 데이터를 읽어 목록을 자동 갱신 — 운영자가 수동으로 명세서를 업데이트하지 않아도 됨
- **기존 Properties 메뉴 대체**: 전환 기간 동안 기존 Properties 메뉴와 병행 운영 후 완전 전환

### Phase 2 예정 (What's out)

- 캠페인 소재 탐색 (기획전/쿠폰/메시징 소재) — Phase 2 예정
- Generic Schema 기반 전 소재 공통 필드 고도화 — Phase 2 예정
- 속성 수집 On/Off 직접 제어 — Phase 2 예정
- Stewardship 워크플로우 (데이터 책임자 지정·승인) — Phase 2 예정
- LLM 기반 자연어 탐색 ("최근 구매 안 한 사용자 찾아줘") — Phase 3 예정

- [ ] **사용 가이드 — 마케터용** (영상 5분 이내): 속성 탐색 → 메타데이터 확인 → 오디언스 적용 3단계 시나리오 중심
- [ ] **사용 가이드 — 운영자·엔지니어용** (문서): 수집 상태 확인, 자동 동기화 동작 방식, 신규 속성 감지 후 메타데이터 입력 방법
- [ ] **슬랙 공지 초안**: 전사 채널용(1건) + 마케팅 채널용(1건) — 기존 DA Slack 문의 대신 도구를 쓰도록 유도하는 문구 포함
