## 기본 원칙

GTM 브리프는 마케터, 기획자, 운영 담당자가 읽는 문서다.
개발자나 PM이 아닌 **비기술직 독자**를 항상 기준으로 삼는다.

---

## 언어 규칙

### 사용 금지 단어 (기술 용어)

| 금지 표현 | 대체 표현 |
|---------|---------|
| API 연동 | 데이터 자동 전달 / 실시간 반영 |
| 배치 처리 | 정해진 시간에 자동 처리 |
| DB / 데이터베이스 | 저장소 / 데이터 |
| 파이프라인 | 처리 흐름 |
| 자동화 솔루션 | (구체적 행동 묘사로 대체) |
| 시스템 | (기능 또는 서비스로 대체) |
| 마이그레이션 | 데이터 이전 |
| 스키마 | 구조 / 양식 |

### 권장 표현 패턴

- **행동 중심**: "~할 수 있다" → "~를 한 곳에서 처리한다"
- **비교 중심**: "기존보다 빠르다" → "기존 3단계 → 1단계로"
- **사용자 이익 중심**: "기능이 추가된다" → "담당자가 ~를 더 이상 하지 않아도 된다"

---

## 톤 (Tone)

| 속성 | 설명 |
|------|------|
| 명확함 | 한 문장에 하나의 메시지만 담는다 |
| 실용적 | 추상적 가치보다 구체적 행동 변화를 묘사한다 |
| 솔직함 | Phase 1 한계를 숨기지 않고 명확히 전달한다 |
| 간결함 | One-liner 50자 이내, Key Message 20자 이내 |
