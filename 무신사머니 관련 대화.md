현재 기준으로 머니 관련 데이터가 전달되는 파이프라인은 다음과 같습니다.  
  

> (최초 페이먼츠의 머니데이터를 무신사로 전달하기 위한 히스토리)  
>   
> 무신사 내 타 조직에서 머니 관련 데이터를 페이먼츠 제공 API를 통해 직접 수신하기 어려운 구조였다고 기억합니다.  
> (페이먼츠를 호출해서 데이터를 드리븐 할 조직이 당시 없다고 이해)  
> 따라서 초기 OCMP에서 팀 무신사 데이터를 수집하던 파이프라인 방식을 페이먼츠에도 유사하게 적용하여 데이터를 전달할 수 있도록 파이프라인을 구성하였습니다.  
> 과거 히스토리는 3개월로 사라져 있지만 비공개 채널 채널에서 `장성우님` 과 이와 같은 작업을 했었습니다.

  
  
간략하게 현재 아래 과정으로 기존 데이터가 전달되고 있습니다.  

- CDC처리: 페이먼츠 선불데이터 - DMS - Kinesis > 무신사 OCMP S3 > Databricks
- 복구처리: OCMP에서 데이터 복구 필요시 페이먼츠 S3에서 DB 스냅샷을 가져가도록 되어 있고, 접근 policy 허용되어 있음

  
  
따라서 상기 파이프라인대로 그대로 진행하신다면 아래와 같이 진행하면 될 것 같습니다.  

- 페이먼츠 선불팀에서 요건에 맞는 데이터 적재 개발
- 페이먼츠 DevOps에서 상기 파이프라인 방식으로 대상 테이블 x개 추가
- 무신사 데이터 조직 혹은 OCMP 계정 담당 조직에서 처리

  
  
만약 위 파이프라인이 아닌 API를 호출해서 가져가실 수 있다면,
