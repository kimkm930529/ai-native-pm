# 에픽 의존성 및 날짜 계산 규칙

`epic-architect` 에이전트가 Start/Due Date를 계산할 때 참조하는 규칙 문서.

---

## 1. 기본 공식

```
Start Date = MAX(오늘 날짜, 선행 에픽 Due Date)
Effort Days = LLM 추정값 (아래 가이드 참조)
Due Date = Start Date + CEIL(Effort Days × 1.2)  ← 20% 버퍼 포함
```

날짜는 **영업일(평일) 기준**으로 계산한다 (주말 건너뜀).

---

## 2. 직무별 기본 의존성 패턴

### 일반적인 순서 (참고용, PRD 내용에 따라 변경 가능)

```
① BE 에픽 시작   →  BE API 완료 시점
     ↓ +3 영업일
② FE 에픽 시작   (API 계약 완료 후 병렬 가능한 경우 더 일찍 시작 가능)
     ↓ 동시 시작 가능
③ DS 에픽 시작   (데이터 분석은 일반적으로 BE와 독립적으로 진행)
     ↓ BE/FE 어느 정도 완료 후
④ MLE 에픽 시작  (학습 데이터 / 피처 파이프라인이 BE 완료에 의존하는 경우)
```

### 직무별 선행 에픽 기본 규칙

| 직무 | 일반적인 선행 조건 | 병렬 가능 여부 |
|------|-----------------|-------------|
| BE | 없음 (최우선 시작) | DS와 병렬 가능 |
| FE | BE API 설계 완료 (BE Start + 3일) | DS와 병렬 가능 |
| MLE | BE 데이터 파이프라인 완료 | 일부 피처 개발은 BE와 병렬 가능 |
| DS | 없음 또는 BE 데이터 접근 확보 후 | BE/FE와 병렬 가능 |

---

## 3. 공수 추정 가이드

### 기능 복잡도 기준

| 복잡도 등급 | 판단 기준 | 추정 공수 (영업일) |
|-----------|---------|----------------|
| **간단** | CRUD API ≤ 5개, 정책 없음, 외부 연동 없음 | 3 ~ 5일 |
| **중간** | 비즈니스 로직 있음, API 6~10개, 상태 관리 | 6 ~ 10일 |
| **복잡** | 외부 시스템 연동, 복잡한 상태 머신, 신규 데이터 모델 | 12 ~ 20일 |
| **고복잡** | ML 모델 학습+서빙, 대규모 마이그레이션, 전체 설계 변경 | 21 ~ 30일 |

### 직무별 공수 보정 계수

| 직무 | 보정 계수 | 이유 |
|------|--------|------|
| BE | 1.0× | 기준값 |
| FE | 0.8× | BE API 완료 후 병렬 작업 가능, 의존성 리스크 낮음 |
| MLE | 1.5× | 실험-학습-배포 사이클로 불확실성 높음 |
| DS | 1.2× | 데이터 탐색 시간 포함, 반복 분석 필요 |

**최종 공수 = 복잡도 추정값 × 직무 보정 계수**

## 4. 날짜 계산 예시

### 예시: Campaign Meta Engine (BE + FE + DS)

```
오늘: 2026-03-10 (월요일)

[BE] Campaign Asset API 개발
  → 의존성: 없음
  → 공수: 복잡 (12일) × BE (1.0×) = 12일
  → 버퍼: 12 × 1.2 = 15일 (영업일)
  → Start: 2026-03-10
  → Due: 2026-03-31 (15 영업일 후)

[FE] MATCH 어드민 UI 구현
  → 의존성: BE Start + 3 영업일 = 2026-03-13
  → 공수: 중간 (8일) × FE (0.8×) = 6.4 → 7일
  → 버퍼: 7 × 1.2 = 9일 (영업일)
  → Start: 2026-03-13
  → Due: 2026-03-25

[DS] 타겟팅 실험 설계
  → 의존성: 없음 (데이터 접근 가능 가정)
  → 공수: 간단 (5일) × DS (1.2×) = 6일
  → 버퍼: 6 × 1.2 = 8일 (영업일)
  → Start: 2026-03-10
  → Due: 2026-03-20
```

## 5. 에지 케이스 처리

| 상황 | 처리 방법 |
|------|---------|
| 선행 에픽 Due Date가 과거 | 오늘 날짜를 Start Date로 사용 |
| PRD에 마일스톤이 명시된 경우 | 명시된 날짜를 Due Date 상한으로 설정 |
| 의존성 순환 감지 | 순환 끊고 open_questions에 기록 |
| 공수 추정이 매우 불확실한 경우 | 범위로 표기 (예: "8~12일") 후 중앙값 사용 |
| 외부 API/파트너 의존성 있는 경우 | 공수에 +3~5일 추가 (협의 시간 반영) |
