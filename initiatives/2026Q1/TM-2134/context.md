## 배경 (Background)

### 기존 방식과 문제점

기존에는 모든 유저에게 동일한 할인율을 일괄 지급하는 **Mass 지급** 방식을 운영했다.
운영은 단순하지만 실제 구매 행동 변화에 대한 데이터 검증이 부재하여 이해관계자 모두가 비효율을 겪고 있었다.

| 주체 | 문제 상황 |
|------|---------|
| 커머스 | 쿠폰이 실제 구매를 유발했는지 불분명하며, '어차피 살 사람'에게도 비용이 지출됨 |
| 파트너 | 타겟팅 수단 부재로 쿠폰 효과를 확신하지 못해 최소 범위로만 운영함 |
| 유저 | 무분별한 노출로 피로도 증가하며, 정작 필요한 혜택은 부족함 |

### 핵심 문제: Right User & Right Coupon 부재

- **Right User 부재**: 쿠폰 없이도 구매했을 유저를 식별하지 못해 불필요한 비용 발생
- **Right Coupon 부재**: 유저마다 다른 할인율 임계점을 무시하고 동일 할인율 적용 → 자극 부족 또는 비용 낭비

### 쿠폰 현황 수치

- 사용 비중: 주문 건수의 약 **63%**, 주문액의 약 **69%** 가 쿠폰 사용 (업셀링 도구로 작동)
- 비용 부담: **자사 77.7%** vs 파트너 22.3%
- ROI: 자사(7.7) 대비 파트너 부담 쿠폰(10.1)의 효율이 더 높음
- 특이사항: 장바구니 쿠폰 단독 사용 주문은 거의 0% → 독립적 구매 유인 효과 미미

## 핵심 개념 (Key Concepts)

| 개념 | 설명 |
|------|------|
| 증분 효과 (Incremental Effect) | 쿠폰이 없었다면 발생하지 않았을 '추가 구매'를 측정 |
| 선택 편향 (Selection Bias) | 쿠폰 사용 유저만 분석 시 구매 의향이 높은 유저 쏠림으로 결과 왜곡 |
| 반사실적 추론 (Counterfactual) | '이 유저에게 쿠폰을 주지 않았다면?'을 답하기 위해 대조군 설계 필수 |
| ITT vs ATT | 실험군 전체 분석(ITT) vs 실제 사용 유저만 분석(ATT)의 관점 차이 |
| Bayesian Smoothing | 데이터가 부족한 개인 유저의 CVR을 추정하기 위해 집단 평균을 활용 |

## 1차 실험 결과 (2025.12)

### 실험 설계

- 대상: 약 210만 명 (앱 푸시 채널)
- 그룹: 대조군(10% 일괄) / 실험군 A(비용 최소화) / 실험군 B(GMV 최대화)

### 결과 요약

| 관점 | 결과 |
|------|------|
| ITT (전체 실험군) | 세 그룹 간 GMV 차이 거의 없음 → 타겟팅 효과 확인 어려움 |
| ATT (쿠폰 사용자) | 실험군이 대조군 대비 낮은 할인율로 더 높은 비용 효율 달성 |

### 한계점

- 대조군 쿠폰 미발급 사고 발생
- 앱 푸시 채널 특성상 낮은 인지율 및 모수 부족
- 선택 편향 개입 (쿠폰 사용 의향 있는 유저만 반응)
- 홀드아웃 그룹 부재 → Incremental Effect 측정 불가

## 2차 실험 설계 (2026.03 예정)

### 주요 변경사항

- 채널: 앱 푸시 → **앱 메인 빅배너** (인지율 및 모수 확대)

## 핵심 가설 (Key Hypotheses)

1. 차등 할인율 타겟팅이 일괄 10% 쿠폰 대비 GMV를 높일 수 있다 (GMV Upside 증명 목표)
2. CRM 스코어 기반 우선순위 배정으로 Sure Thing(어차피 살 사람) 비율을 줄일 수 있다
3. 앱 메인 빅배너로 채널 변경 시 1차 실험 대비 유효 모수가 충분히 확보된다

---

## 증명 가능 / 불가능 범위 (Scope)

**증명 가능 (In Scope)**:
- 차등 할인율이 일괄 10% 쿠폰보다 GMV를 높일 수 있는가 (GMV Upside)

**현재 증명 불가 (Out of Scope)**:
- 쿠폰 자체가 없었을 때와의 비교 (Incremental Effect) — 홀드아웃 그룹 부재
- 최적 할인율과 등급 쿠폰 간 딜레마 해소
- 학습 데이터 오염 문제 (Sure Thing 혼재)
