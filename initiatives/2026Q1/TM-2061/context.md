# TM-2061: Campaign Meta Engine (Phase 1) — 컨텍스트

> **에이전트 참조용**: 이 이니셔티브와 관련된 질문·작업이 들어오면 이 파일을 먼저 읽어 배경을 파악한다.

## 배경 (Background)

무신사의 마케팅 운영은 프로모션 푸시(11시 세일즈푸시, 17시 프로모션푸시), 라이브 푸시 등 다양한 캠페인의 타겟팅을 **수기로 설정하고 스프레드시트(엑셀)로 관리**하고 있다. 이는 두 가지 구조적 병목을 초래한다.

| 문제 | 상세 |
|------|------|
| **수기 타겟팅 비효율** | 마케터가 매일 캠페인을 수동 등록·관리 → 운영 부담. 스프레드시트 기반은 동기화 지연 등 제어 한계 존재 |
| **비표준화된 콘텐츠 메타데이터** | 무신사 CMS(CPCMS) 내 콘텐츠 데이터가 통일된 메타데이터 기준 없이 관리 → ML 모델링 시 콘텐츠 간 적합도·우선순위 비교 불가 → 타겟팅 최적화 제약 |

현재 자동화로 운영 중인 캠페인(14시 개인화푸시)조차 CMS 콘텐츠 메타데이터 비표준화로 ML 성능이 제한적인 상태다.

## 시스템 구조 — 핵심 모듈

```
Campaign Meta Engine
├── Sync          : MATCH ↔ Braze 연결 어댑터 (이미 구축 완료)
│                   캠페인 설정을 Braze에 전달, API-triggered 실행 방식
├── Send Group    : 특정 시점에 발송될 '발송 집단 단위' (코호트 + 소재)
│                   하나의 Sync → 하나 이상의 Send Group
│                   (일회성: 1개, 반복: 다수 — 예: 세일즈푸시 날짜별 그룹)
├── Campaign Asset: 홍보 대상(기획전, 상품 등)의 표준화된 정보 단위
│                   URL 기반으로 CMS에서 메타 정보 자동 수집·정규화
│                   소재 범위 (Phase 1): 기획전, 매거진, 라이브 (상품 단위 제외)
├── Message Template: 자동화 캠페인 메시지 생성 규칙 (Phase 1: 수동 입력만)
└── Conversion    : 광고 코드(Tracking Code) 기반 성과 측정 표준화
                    Campaign Asset 단위로 광고코드 자동 생성·관리
```

## 핵심 기능 요구사항 (Phase 1)

### 1. Sync 생성 및 Braze 연동
- Braze 캠페인 ID 입력 → API로 캠페인 정보 실시간 조회·검증
- 코호트 설정: **자동 코호트** (마케팅 수신 동의 전체, MATCH 자동 산출) vs **사용자 지정 코호트**
- 발송 주기/기간 설정 ('종료일 없음' 옵션 포함)
- Phase 1: 개인화 변수 설정 제거 (추후 Message Template 영역에서 도입)

### 2. Campaign Asset 일괄 등록 (엑셀 업로드)
- 엑셀 템플릿 다운로드 → 필드 입력 후 업로드
- 시스템이 Landing URL 기반으로 CMS(CPCMS) 연동 → 브랜드, 할인율, 종료일 메타데이터 자동 수집 및 표준 스키마로 정규화
- CMS 미등록 링크인 경우: 사용자가 소재 정보 직접 보완 또는 브랜드 정보 등록
- 업로드 최대 1,000행 / 전체 중 하나라도 오류 시 전체 업로드 취소 (부분 성공 불가)

**엑셀 템플릿 주요 필드**

| 필드 | 필수 여부 | 형식 / 상세 |
|------|---------|-----------|
| 발송일 (send_dt) | 필수 | YYYY-MM-DD |
| 발송 대상 (target) | 필수 | 전체 / 남성 / 여성 |
| 우선순위 (priority) | 선택 | 정수 (낮을수록 높음) |
| 광고 식별자 (ad_code) | 필수 | APSCMCD 형식 |
| 브랜드 (brand_id) | 선택 | 미입력 시 landing_url 기준 자동 적용 |
| 캠페인명 (braze_campaign_name) | 필수 | Braze 전달 캠페인명 |
| 타이틀 (title) | 필수 | 앱 푸시 타이틀 |
| 본문 (contents) | 필수 | 앱 푸시 본문 |
| 랜딩 URL (landing_url) | 필수 | 소재 랜딩 URL |
| 이미지 URL (image_url) | 필수 | 앱 푸시 썸네일 |

> 기존 세일즈푸시 템플릿 대비 제거된 칼럼: send_time, send_day_cd, category, push_url, setting_yn

### 3. 광고코드 자동 생성 (Tracking Code)
- 소재 입력 시 UTM 파라미터 포함 Tracking Code 자동 생성:
  `?utm_source=app_push&utm_medium=cr&utm_content=mf&utm_campaign={ad_code}&source={ad_code}`
- Phase 1: 수동 메시지 입력만 제공

### 4. 모델링 및 발송
- 등록된 소재·캠페인 목적에 따라 ML 모델링 수행 (21시~07시 처리)
- 모델링 완료 후 [발송 대상] 컬럼에 할당 유저 수 표시
- 상태: **발송 예정 → 발송 완료 / 미발송**

## 핵심 가설 (Key Hypotheses)

1. Campaign Asset 표준화 → ML 모델링 성능 개선 → 매칭 정확도 상승 → **Push CTR 개선**
2. MATCH 어드민 직접 제어 방식 → 스프레드시트 대비 동기화 지연 없는 **즉각적·정확한 운영** 가능
3. Campaign Adoption Rate 50%+ 달성 시 → Frequency Capping·캠페인 간 우선순위 등 **고도화 최적화를 위한 기반 마련**

## 주요 제약사항 (Constraints)

- Braze를 통해서만 메시지 발송 (Webhook 기반 수신 동의 필터링, Frequency Capping, Re-Eligible 규칙 그대로 준수)
- Phase 1: 소재 등록 방식은 수동 등록만 제공 (자동 소재 선정은 Phase 2)
- 메시지 개인화 변수는 Phase 1에서 미제공
- 데이터 스냅샷 방식: 등록 시점 데이터 보존, 이후 CMS 원본 변경 시 즉시 동기화 불가 (Phase 1)
