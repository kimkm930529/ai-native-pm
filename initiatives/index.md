## 폴더 구조 규칙

```
initiatives/
├── index.md                     ← 이 파일 (분기별 현황 인덱스)
├── _template/                   ← 새 이니셔티브 추가 시 복사할 템플릿
└── 2026Q1/
    └── {TICKET-ID}/
        ├── meta.json            ← 티켓 메타데이터 (ID, 상태, 기간, 관련 Space)
        ├── context.md           ← 배경·목표·핵심 가설·성공 지표
        ├── decisions.md         ← 의사결정 로그
        ├── references.md        ← 관련 Confluence/Jira/Slack 링크
        └── output/              ← 이 이니셔티브로 생성된 산출물 (PRD, 분석 등)
```

## 에이전트 활용 가이드

- **조회 요청** 시: 관련 이니셔티브 폴더의 `context.md` + `references.md`를 먼저 참조
- **문서 업로드** 시: `meta.json`의 `confluence_space`를 기본 타겟 Space로 사용
- **의사결정 기록** 시: 해당 이니셔티브의 `decisions.md`에 로그 추가
- **새 이니셔티브 추가** 시: `_template/` 폴더를 복사하여 `2026Q1/` 하위에 생성
