## (1) 목표

### 기능 목표 요약

- 무신사 CMS(Contents)와의 연동을 통해 캠페인 운영에 필요한 콘텐츠 메타데이터를 자동으로 수집·정규화하고, 이를 MATCH의 표준 캠페인 스키마로 관리함으로써 **Content-based CRM 운영의 구조적 기반을 구축합니다.**
- Campaign Asset을 중심으로 콘텐츠–메시지–성과 측정의 기준을 일관되게 정립하여, 기획전·콘텐츠 단위로 캠페인을 운영하고 최적화 할 수 있는 표준 메타데이터셋을 마련합니다.
- 엑셀 기반으로 분산되어 운영되던 캠페인 등록·발송·성과 관리를 MATCH 내 표준 캠페인 구조로 통합합니다.

### 비즈니스 목표 요약 — Scope (의사결정 분기표)

|-----------------|----------------|----------------------|

## (2) 기능 상세

### [a] Sync 생성 및 관리

**LNB 경로**: Activation > 마케팅 > 캠페인 연동

**목록 표시 항목**: Sync 명칭 / Braze 캠페인명 / 코호트 방식 / 발송 주기 / 생성일 / 최근 발송일

#### [a]-2. Sync 생성 모달

| No. | 구분 | 상세 정책 |
|-----|------|---------|
| 1 | **모달 진입** | [신규 캠페인 연동] 클릭 시 모달 노출. 모든 필드는 빈 상태로 초기화 |
| 2 | **Braze 캠페인 ID 입력** | 필수 항목. 입력 후 포커스 아웃 시 Braze API 실시간 호출하여 유효성 검증. [유효] API 검증 성공: Braze 캠페인명 필드에 조회된 명칭 자동 매핑(읽기 전용). [무효] API 검증 실패: 인라인 오류 메시지 표시 — "입력한 Braze 캠페인 ID를 찾을 수 없습니다. Braze 콘솔 > 캠페인 > 캠페인 ID를 확인해주세요." |
| 3 | **Braze 캠페인명** | Braze 캠페인 ID 검증 성공 시 자동 매핑. 참조용 메타데이터로 활용되며 수정 불가 |
| 4 | **코호트 설정** | **자동 코호트**: 발송 대상 = 마케팅 수신 동의 전체 사용자로 자동 고정. [코호트 검색] 입력 필드 비활성화 (Phase 1 범위). **사용자 지정**: [코호트 검색] 필드 활성화. 사전 정의된 코호트 목록에서 검색 후 선택. 해당 Sync는 MATCH 타겟팅 최적화 미적용 |
| 5 | **발송 주기·기간** | 일회성 / 반복 여부 설정. **'종료일 없음' 옵션 제공** — 생성일 기준 365일 후 등록자에게 '운영 현황 검토 알림' 슬랙 메시지 발송. 캠페인 자동 중단은 없으며 마케터가 수동 종료 또는 유지 결정 |
| 7 | **개인화 변수 제외** | Phase 1에서 제외. Message Template 및 자동화 영역에서 단계적 도입 예정 |
| 8 | **[생성] 버튼 활성 조건** | 자동 코호트 선택 후: ① Braze 캠페인 ID 검증 완료 ② 코호트 설정 완료 ③ 발송 주기·기간 설정 완료 — 3가지 조건 모두 충족 시 활성화. 사용자 지정 선택 시: 버튼 비활성화 |
| 9 | **Sync 생성 완료 처리** | 생성 성공 시 모달 자동 닫힘 → 신규 생성된 Sync 상세 페이지로 자동 이동 |

#### [a]-3. Sync 상세 정보 (상단 영역)

- **Sync 명칭 강조 표시** (예: `251205_APSCMCD_세일즈푸시(11시)`)
- **수정 버튼**: Sync의 기본 설정(코호트 방식, 발송 주기 등)을 수정하는 Sync 모달로 이동. 이미 생성된 캠페인과 과거 발송 이력에는 영향 없음
- **삭제 버튼**: Sync 영구 삭제. 아직 발송되지 않은 모든 항목은 즉시 중단. 확인 모달 표시: "발송 예정 N건이 즉시 취소됩니다. 삭제 후 복구가 불가합니다. 계속하시겠습니까?"
- **캠페인 기본 정보 (Master Info)**: Braze 캠페인 ID, Braze 캠페인명, 오디언스, 발송 방식, 소재 등록 방식(수동/자동), 타이틀/본문 등록 방식

### [b] CMS 소재 연동 공통 정책

#### URL 유형별 처리 방식

| No. | 유형 | URL 패턴 | 처리 방법 |
|-----|------|---------|---------|
| 1 | **기획전** | `/campaign/` 경로 포함 | HTML 파싱 (`html_code` 기반 메타 추출) |
| 2 | **매거진·콘텐츠** | `/contents/` 경로 포함 | URL에서 콘텐츠 ID 추출 → content-API 호출 → 테이블 조회 |
| 3 | **라이브** | `/live/` 경로 포함 | URL 내 고유 키값 추출 → 라이브 조회 API 호출 |
| 4 | **브랜드 페이지** | `/brands/` 경로 포함 | 브랜드 페이지 파싱 → 브랜드 정보 수집 |
| 5 | **미지원** | 위 패턴 미해당 (무퀴즈·래플 등) | 수동 입력 모드 전환 |

#### [b]-3. 수동 입력 모드 (미지원 URL 또는 API 실패 시)

| No. | 구분 | 상세 정책 |
|-----|------|---------|
| 1 | **진입 조건** | ① 미지원 URL 유형 판별 시 / ② 지원 유형이나 API 응답 없음(미등록 소재) / ③ API 타임아웃 3초 초과 |
| 2 | **수동 입력 필수값** | `brand_id` 직접 입력 필수 |
| 3 | **안내 문구** | "자동 연동이 지원되지 않는 URL입니다. 브랜드 정보를 직접 입력해주세요." |
| 4 | **엑셀 등록 시 처리** | 해당 행에 "미등록 소재" 뱃지 표시 + `brand_id` 입력 필드 활성화. 미입력 시 발송 예정 전환 불가 |
| 5 | **단일 등록 시 처리** | 등록 모달 내 수동 입력 모드로 전환. `brand_id` 미입력 시 [등록] 버튼 비활성 |

#### [b]-4. 연동 데이터 수정 정책

| No. | 구분 | 상세 정책 |
|-----|------|---------|
| 1 | **자동 바인딩 후 수정 허용** | CMS에서 자동 수집된 값(`image_url`, `brand_id`, 노출기간 등)은 마케터가 직접 수정 가능. 수정값이 CMS 원본보다 우선 적용됨 |
| 2 | **`brand_id` 수동 우선 정책** | 엑셀의 `brand_id` 컬럼에 직접 입력값이 있으면 CMS 자동 수집값을 덮어쓰지 않음 |
| 3 | **스냅샷 고정** | 등록 완료 후에는 CMS 원본이 변경되어도 Campaign Asset에 즉시 동기화되지 않음 (to-be: [CMS 최신 정보로 갱신] 버튼 제공 예정) |
| 4 | **등록 완료 후 수정** | 발송 전까지 캠페인 상세 화면에서 수정 가능. 단, `landing_url` 변경 시 CMS 연동 재호출 여부는 개발팀 확인 필요 *(Open)* |

#### [b]-5. 연동 데이터 예외 정책

만료된 CMS 소재 선택 시: 소재의 CMS 기준 종료일이 발송일 이전인 경우, '해당 소재의 전시 기간이 종료되었습니다. 계속 사용하시겠습니까?' 경고 표시 → 차단하지 않으며 마케터가 최종 판단

### [g] 캠페인 상세

**목적**: 특정 캠페인의 상세 설정 값과 실제 발송될(또는 발송된) 콘텐츠의 최종 결과물을 확인하고 관리함.

**구성**: 정보 입력/조회 영역(좌측)과 실시간 미리보기 영역(우측)으로 구분된 2단 레이아웃.

#### 3.2 콘텐츠/소재 정보

- **소재 범위 (Phase 1)**: 기획전, 매거진, 라이브 (※ 상품 단위는 이번 범위에서 제외)
- **검색 필터**: 콘텐츠 유형(기획전, 매거진, 라이브), 브랜드, 키워드
- **소재 선택 및 매핑**: 리스트에서 특정 소재 선택 시, 해당 소재의 **표준 메타데이터**(광고코드 후보, 랜딩 URL, 이미지 등)가 캠페인 설정 영역에 자동 바인딩됨
- **데이터 관리 방식**: **스냅샷(Snapshot)** 방식 채택 — 등록 시점의 데이터를 보존하며, 이후 CMS 원본이 변경되어도 본 캠페인 정보에 즉시 동기화되지 않음 (Phase 1)

## (5) Appendix

### 용어 및 개념

| 레이어 | 용어 | 정의 |
|--------|------|------|
| **인프라** | **MATCH** | 무신사 CRM 자동화 플랫폼. 캠페인 의사결정·스케줄링·ML 모델링을 담당하며 Braze로 발송을 트리거한다. |
| | **Braze** | 마케팅 자동화 플랫폼. 실제 앱 푸시 발송을 담당하는 외부 시스템. MATCH의 지시를 받아 실행한다. |
| | **CMS (CPCMS)** | 무신사 콘텐츠 관리 시스템. 기획전·매거진·라이브 등 콘텐츠의 메타데이터 원본 저장소. |
| | **Content-API** | CMS에서 Landing URL 또는 소재 ID 기반으로 메타데이터(브랜드·이미지·노출기간 등)를 조회하는 API. |
| **데이터 모델** | **Sync** | MATCH와 Braze 캠페인을 연결하는 **어댑터 단위**. "누구에게 / 언제 / 어떤 반복 주기로" 발송할지를 정의한다. 하나의 Sync 아래에 N개의 Campaign Asset(발송 row)이 등록된다. |
| | **Campaign Asset** | 개별 발송 건의 **표준 콘텐츠 스키마 단위**. Landing URL 기반으로 CMS에서 메타데이터(brand_id·image_url·노출기간 등)를 수집·정규화하여 MATCH DB에 적재한 결과물. |
| 코호트 | **코호트 (Cohort)** | 캠페인 발송 대상으로 정의된 유저 그룹. |
| | **자동 코호트** | Phase 1 범위: 마케팅 수신 동의 전체 사용자. MATCH가 코호트를 자동 관리. |
| | **사용자 지정 코호트** | 마케터가 사전에 정의한 특정 유저 그룹을 직접 선택. 이 경우 MATCH 타겟팅 최적화 미적용. |
| 소재 식별자 | **ad_code** | 캠페인 추적용 고유 식별자. `APSCMCD{숫자}` 형식. |
| | **brand_id** | 소재에 연동된 브랜드 식별자. Content-API로 자동 수집, 미등록 소재는 마케터가 직접 입력. |
| | **UTM 파라미터** | `?utm_source=app_push&utm_medium=cr&utm_content=mf&utm_campaign=APSCMCD{코드}&source=APSCMCD{코드}` |
| | **landing_url** | 앱 푸시 클릭 시 이동하는 원본 URL. UTM 이전 순수 콘텐츠 주소. |
| | **push_url** | UTM 파라미터가 자동 조합된 최종 랜딩 URL. |
| 데이터 정책 | **스냅샷 (Snapshot)** | Campaign Asset 등록 시점의 CMS 메타데이터를 MATCH DB에 고정·보존. 이후 CMS 원본 변경 시 즉시 반영되지 않음. |
| | **미등록 소재** | Content-API 조회 결과 CMS에 없는 Landing URL. brand_id 직접 입력 후 등록 가능. |
| | **Global Optimizer** | Phase 2 예정 기능: ML 기반 소재·발송 최적화. |

| 구분 | As-is (Databricks) | To-be (MATCH) | 절감 |
|------|-------------------|--------------|------|
| 소재 준비 (URL·UTM 조합) | 40~80분 | 10~20분 | **~70% 절감** |
| 타겟팅 설정 | 45~90분 | 5~10분 (Sync 최초 생성, 이후 0분) | **~90% 절감** |

| **총계** | **125~265분** | **23~45분** | **약 75~83% 절감** |

**핵심 개선점**: Databricks에서는 마케터가 코드(쿼리·페이로드)를 직접 작성해야 하지만, MATCH에서는 Excel 입력만으로 동일한 API Triggered 발송이 자동 실행됨. CMS 메타데이터 수집, UTM 생성, ML 모델링이 모두 자동화됨.
