# **(1) 목표**
| 
**기능 목표 요약**
 | 
- 
무신사 CMS(Contents)와의 연동을 통해 캠페인 운영에 필요한 콘텐츠 메타데이터를 자동으로 수집·정규화하고, 이를 MATCH의 표준 캠페인 스키마로 관리함으로써 **Content-based CRM 운영의 구조적 기반을 구축합니다.**

- 
Campaign Asset을 중심으로 콘텐츠–메시지–성과 측정의 기준을 일관되게 정립하여, 기획전·콘텐츠 단위로 캠페인을 운영하고 최적화 할 수 있는 표준 메타데이터셋을 마련합니다. 엑셀 기반으로 분산되어 운영되던 캠페인 등록·발송·성과 관리를 MATCH 내 표준 캠페인 구조로 통합합니다.

**배경**
 | 
- 
현재 무신사의 마케팅 운영은 프로모션 및 라이브 푸시 등의 타겟팅을 수기로 설정하고 엑셀 기반으로 관리하고 있어 운영 효율과 확장성 측면에서 한계에 직면해 있습니다.

- 
기존의 스프레드시트 기반 관리 구조는 동기화 지연 등 운영 관리의 제약이 크므로, 어드민 직접 제어 방식으로 전환하여 즉각적이고 정확한 운영 관리가 필요합니다.

- 
무신사 CMS 내 콘텐츠 메타데이터가 비표준화되어 있어 ML 기반의 최적화와 소재 간 적합도 비교가 어려운 구조적 병목 현상을 해결해야 합니다.
 |
| 
구축 목표
 | 
- 
**Campaign Meta Engine 구축**: 캠페인 설정부터 발송, 성과 측정까지 이어지는 End-to-End 캠페인 파이프라인을 시스템화합니다.

- 
**데이터 표준화**: 모든 캠페인 소재를 MATCH 표준 스키마로 정규화하여 제공함으로써 ML 모델링 성능을 높이고 정교한 소재 경쟁 구조를 설계합니다.

- 
**운영 자동화 기반 마련**: 사용자가 매일 소재를 직접 고르고 메시지를 작성하지 않아도 되는 자동화 캠페인 운영 체계를 단계적으로 완성합니다.

# **(3) 정책 및 상세 기획**

## 3-1. 용어 정의
**Conceptual Model**

- 
**Sync:** MATCH와 Braze 캠페인을 연결하는 어댑터.

- 
**Send Group:** 특정 시점에 발송될 '발송 집단' 단위. (코호트 + 소재)

- 
**Campaign Asset:** 홍보 대상(기획전, 상품 등)의 표준화된 정보 단위. URL 기반으로 CMS에서 메타 정보를 가져옴.
