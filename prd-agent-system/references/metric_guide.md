# 지표 정의 가이드 — PRD Metrics Reference

## 지표 프레임워크 개요

PRD의 모든 지표는 아래 3계층 구조로 정의한다.

```
North Star Metric (1개)
    └── Primary Metrics (2~4개)
            └── Guardrail Metrics (1~3개)
```

---

## 1. North Star Metric

**정의:** 제품이 궁극적으로 추구하는 단 하나의 지표.
비즈니스 성장과 사용자 가치를 **동시에** 반영해야 한다.

**설정 기준:**
- 조직 전체가 한 방향으로 정렬(Align)될 수 있는가?
- 6~12개월 단위로 측정 가능한가?
- 허수 지표(Vanity Metric)가 아닌 실질 사용자 가치를 측정하는가?

**도메인별 예시:**

| 도메인 | North Star 예시 | 이유 |
|--------|----------------|------|
| 멤버십 | 활성 유료 멤버 수 (M-MAU) | 장기 고객 가치 및 구독 매출과 직결 |
| 커머스 | 구매 전환율 (Session → Purchase) | 트래픽 대비 실질 매출 성과 측정 |
| 앱 푸시 | 푸시 수신 후 24시간 내 구매율 | 채널 효율성과 매출 기여 동시 반영 |
| 검색/추천 | 검색 후 상품 클릭률 | 발견 경험의 질 측정 |
| 리텐션 | 월 2회 이상 구매 고객 비율 | 충성 고객 형성 여부 판단 |

---

## 2. Primary Metrics

**정의:** North Star를 달성하기 위한 중간 목표 지표.
각 지표는 North Star에 **인과적으로** 연결되어야 한다.

**설정 기준:**
- North Star가 상승할 때 함께 상승하는가? (상관관계 검증 필요)
- 2~4주 단위로 측정 및 피드백 가능한가?
- 담당 팀이 직접 영향을 줄 수 있는가?

**유형별 분류:**

| 유형 | 설명 | 예시 |
|------|------|------|
| Acquisition | 신규 유저 유입 | 신규 회원 가입 수, 앱 설치 수 |
| Activation | 핵심 가치 최초 경험 | 첫 구매 완료율, 온보딩 완료율 |
| Retention | 재방문 / 재구매 | D7 리텐션, 월 2회 이상 구매자 비율 |
| Revenue | 매출 기여 | 평균 주문 금액(AOV), 구독 전환율 |
| Engagement | 기능 활성화 정도 | 기능 클릭률, 세션 내 이용 빈도 |

---

## 3. Guardrail Metrics

**정의:** Primary Metrics를 개선하는 과정에서 의도치 않게 악화되면 안 되는 지표.
한계선(Threshold)을 초과하면 롤백 또는 긴급 검토가 필요하다.

**설정 기준:**
- 이 지표가 악화되면 사용자 경험 또는 비즈니스에 치명적인가?
- Primary Metrics 개선과 Trade-off 관계인가?

**공통 Guardrail 예시:**

| 지표 | 설명 | 한계선 기준 예시 |
|------|------|----------------|
| 앱 크래시율 | 앱 강제 종료 비율 | < 0.5% |
| CS 신고율 | CS 접수 건수 / 활성 유저 | 전주 대비 < +20% |
| 푸시 수신 거부율 | 알림 비활성화 사용자 증가율 | 전주 대비 < +2%p |
| 장바구니 이탈률 | 장바구니 → 결제 포기 비율 | < 75% |
| 페이지 로딩 시간 | P95 로딩 시간 | < 3초 |
| 결제 오류율 | 결제 시도 대비 오류 발생 비율 | < 1% |

## 4. 지표 정의 템플릿

PRD에 지표를 기술할 때 아래 형식을 사용한다:

```markdown
## 성공 지표

### North Star Metric
- **지표명:** {지표명}
- **정의:** {분자 / 분모 또는 산식}
- **측정 주기:** {주간 / 월간}
- **목표값:** {론칭 후 N개월 내 X% 개선}
- **비즈니스 연결:** {이 지표가 왜 가장 중요한가}

### Primary Metrics

1. **{지표명}**
   - 정의: {산식}
   - 목표: {기준치(Baseline) 대비 X% 개선}
   - 비즈니스 연결: {North Star에 영향을 주는 인과 관계}

### Guardrail Metrics

1. **{지표명}**
   - 정의: {산식}
   - 한계선: {이 수치 이하/이상이면 롤백 또는 긴급 검토}
   - 모니터링 주기: {일별 / 주별}
```

## 5. 지표 설정 시 흔한 실수

| 실수 유형 | 나쁜 예 | 개선 방안 |
|----------|--------|----------|
| 허수 지표(Vanity Metric) | 페이지 뷰 수, 다운로드 수 | 구매 전환율, 재구매율 등 실질 행동으로 대체 |
| 측정 불가 지표 | "사용자 만족도", "브랜드 인지도" | NPS 점수, 재구매율 등 정량화 |
| 상호 모순 지표 | 신규 가입 수 & 이탈률 동시 개선 | 우선순위 결정 후 한 방향 선택 |
| 단기 지표만 나열 | 일 활성 사용자 수(DAU)만 측정 | DAU + 30일 리텐션 균형 있게 구성 |
| 측정 시점 불명확 | "전환율 개선" | "론칭 후 4주 이내 전환율 기준치 대비 +5%" |

## 6. 지표 검증 질문 (Product Strategist가 Metric Proposal 전 자문)

1. 이 지표가 개선되면 정말 비즈니스 성장으로 이어지는가?
2. 이 지표를 개선하기 위해 팀이 직접 취할 수 있는 행동이 있는가?
3. 이 지표를 조작(게이밍)하여 허수로 개선할 가능성은 없는가?
4. 현재 데이터 인프라로 측정이 가능한가?
5. 경쟁 지표(Trade-off) 관계인 Guardrail을 설정했는가?
