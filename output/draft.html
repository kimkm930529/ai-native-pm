1. Executive Summary

MATCH AI Assistant는 1Q PoC에서 Page0(전사 하이레벨 지표) 기반 분석의 유의미성을 확인했다. 2Q에는 DI 조직이 2026 Q1에 Production 배포한 Semantic Layer(Layer0 SSOT)를 기반으로 마케팅 도메인 L1/L2 마트까지 학습 범위를 확대하고, 오디언스를 29CM 마케팅 실행팀으로 넓혀 분석 리드타임 50% 단축을 목표로 한다. Semantic Layer의 CATC(데이터 품질) SLA를 AI 응답 신뢰도 제도적 기반으로 활용하여, 마케터가 잘못된 인사이트를 의사결정에 반영하는 리스크를 구조적으로 통제한다.

2. Problem Definition

Context

  • 1Q PoC(Phase 1)에서 Page0 데이터 기반 분석이 유의미한 유스케이스를 창출함을 확인 — 1Q AI Assistant Use Case
  • 현재 마케팅 실행팀은 캠페인 성과 분석·오디언스 탐색 시 DA팀 쿼리 작성 요청 또는 BI 툴 직접 조작 → 의사결정까지 리드타임 발생
  • DI 조직이 2026 Q1에 Semantic Layer(Layer0) Production 배포 완료 → 신뢰할 수 있는 SSOT 기반 확보됨
  • 기존 AI Assistant는 Page0(전사 하이레벨 집계 지표) 학습에만 국한 → 마케팅 실행 수준(캠페인·CRM·퍼포먼스)의 세분화 분석 불가

Pain Point

Pain 1. 분석 리드타임 병목

  • 마케팅 실행팀이 캠페인 성과·오디언스 세그먼트 분석 요청 시 현재 평균 리드타임: [현재 리드타임 — PM 기입. 예: 요청 접수 → 결과 전달까지 평균 N시간]
  • 병목 구간: 요청 접수 대기 + DA 쿼리 작성 + 결과 검증 (추정 전체 시간의 80%)

Pain 2. 마케팅 도메인 데이터 접근성 부재

  • Page0는 전사 하이레벨 집계 지표만 포함 → 캠페인별·오디언스 세그먼트별 드릴다운 불가
  • 퍼포먼스 광고, CRM, 오디언스 세그먼트 등 마케팅 도메인 마트가 AI 학습 범위 밖에 존재

3. Proposed Solution

Solution Overview

DI 조직의 Semantic Layer(Layer0 SSOT)를 기반으로 마케팅 도메인 L1/L2 마트까지 AI Assistant 학습 범위를 확장하여, 29CM 마케팅 실행팀이 자연어로 캠페인·오디언스·퍼포먼스 인사이트를 즉시 획득할 수 있도록 고도화한다.

Why Semantic Layer 연동이 핵심인가

DI 조직이 2026 Q1에 Production 배포한 Semantic Layer는 다음을 보장한다:

Semantic Layer 기능AI Assistant에 주는 가치
SSOT(Single Source of Truth) — 팀마다 다른 GMV 집계 로직 문제 해소 AI가 단일하고 검증된 데이터 원천에서만 인사이트 생성 → 응답 정합성 확보
CATC Framework (Completeness·Accuracy·Timeliness·Consistency) — Row Count Drift ≤1%, Freshness ≤7200초 SLA AI 응답에 데이터 품질 SLA 상태를 병기하여 마케터가 응답 신뢰도를 직접 판단 가능
TVF(Table-Valued Functions) 패턴 — period(이벤트)/snapshot(상태) 시계열 분석 표준화 1Q PoC 유스케이스(신규 가입자 급등 원인 분석, 고활성 고객 시계열 트렌드)와 같은 시계열·코호트 분석을 표준 패턴으로 지원
Grace Period(D+30) — 주문 상태 변경 등 비즈니스 라이프사이클 자동 반영 최신 데이터의 오류를 AI가 그대로 학습하는 위험 최소화

참고: Airbnb Minerva — Semantic Layer를 AI 분석의 선행 기반으로 구축한 대표 사례. 지표 정의 중앙화로 팀별 숫자 불일치를 해소하고, NL Query 인터페이스를 그 위에 올린 구조로 본 과제와 동일한 아키텍처.
