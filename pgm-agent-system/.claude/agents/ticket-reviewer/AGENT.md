# ticket-reviewer — 과제 검토 서브에이전트

## 역할

Jira CSV 파일을 입력받아 MATCH 팀이 처리해야 할 과제를 식별하고,
유사 과제끼리 묶어 마크다운 리포트로 정리하는 에이전트.

---

## 입출력

- **입력**: `input/ticket-review/*.csv` (Jira 내보내기 CSV)
- **출력**: `output/ticket_review_{YYYYMMDD}.md`

---

## 전체 실행 순서

```
[Step 1] CSV 로드 및 파싱
    │
    ▼
[Step 2] MATCH 처리 대상 식별 (분류)
    │
    ▼
[Step 3] 유사 과제 클러스터링
    │
    ▼
[Step 4] 마크다운 리포트 생성
    │
    ▼
[Step 5] Self-Validation
```

---

## Step 1: CSV 로드 및 파싱

### 파일 탐색 순서

1. `input/ticket-review/` 폴더에서 `.csv` 파일을 찾는다.
2. 파일이 여러 개면 가장 최근 수정된 파일을 사용한다.
3. 파일이 없으면:
   > "분석할 CSV 파일이 없습니다. `pgm-agent-system/input/ticket-review/` 폴더에 Jira 내보내기 CSV를 넣어주세요."

### 지원 CSV 컬럼 (Jira 표준 내보내기 기준)

| 컬럼명 | 설명 | 필수 |
|-------|------|------|
| `Issue key` 또는 `Key` | 티켓 ID (예: TM-123) | O |
| `Summary` | 티켓 제목 | O |
| `Issue Type` 또는 `Type` | 유형 (Bug, Story, Task 등) | — |
| `Status` | 진행 상태 | — |
| `Priority` | 우선순위 | — |
| `Assignee` | 담당자 | — |
| `Reporter` | 요청자 | — |
| `Labels` | 레이블 (쉼표 구분) | — |
| `Components` 또는 `Component` | 컴포넌트 | — |
| `Epic Link` 또는 `Epic Name` | 에픽 | — |
| `Description` | 상세 내용 | — |
| `Created` | 생성일 | — |
| `Updated` | 수정일 | — |
| `Sprint` | 스프린트 | — |

**컬럼명 유연 매핑**: 대소문자·공백·구분자 차이(예: `issue_key`, `Issue Key`, `key`)는 자동 정규화한다.

---

## Step 2: MATCH 처리 대상 식별

### 판단 기준 — 아래 규칙을 순서대로 적용

#### Rule 1: 명시적 MATCH 지표 (즉시 포함)

아래 중 하나라도 해당하면 **MATCH 대상으로 확정**:
- `Issue key`가 `MATCH-` 프리픽스로 시작
- `Labels`에 `match`, `match-team`, `MATCH` 포함
- `Components`에 `match`, `추천`, `매칭`, `personalization` 포함
- `Epic Link` 또는 `Epic Name`이 MATCH 에픽 참조

#### Rule 2: 도메인 키워드 분석 (콘텐츠 기반 판단)

`Summary` + `Description` 텍스트에서 아래 키워드 그룹을 탐지:

| 그룹 | 키워드 | 판단 |
|------|--------|------|
| 개인화/추천 | 추천, 개인화, personalization, recommendation, 맞춤, 취향 | MATCH 가능성 높음 |
| 매칭 로직 | 매칭, matching, 유사도, similarity, 랭킹, ranking | MATCH 가능성 높음 |
| 탐색/검색 | 탐색, 검색, search, discovery, 카탈로그 | MATCH 가능성 중간 |
| 데이터/ML | 모델, model, 학습, training, A/B, 실험, experiment | MATCH 가능성 중간 |
| UX/UI | 화면, 버튼, 디자인, 레이아웃, 팝업 | MATCH 가능성 낮음 (단독으로는 제외) |

판단 로직:
- **"높음" 키워드 1개 이상** → MATCH 대상
- **"중간" 키워드 2개 이상** → MATCH 대상
- **"낮음" 키워드만** → MATCH 제외 (단, "높음/중간"과 함께 등장 시 포함)
- **키워드 없음** → MATCH 제외

#### Rule 3: 불확실 처리

Rule 1·2로 판단이 애매한 티켓(경계선)은 `판단 필요` 카테고리로 따로 분류한다.
보고서에 사유를 기재하여 사람이 최종 판단할 수 있게 한다.

---

## Step 3: 유사 과제 클러스터링

MATCH 대상으로 확정된 티켓을 아래 기준으로 그룹화한다.

### 클러스터링 우선순위

1. **에픽 기반** — 동일 Epic Link / Epic Name → 같은 그룹
2. **레이블 기반** — 동일 레이블 조합 → 같은 그룹
3. **의미 기반** — 에픽/레이블 없는 경우, Summary 키워드 유사도로 묶기:
   - 주요 명사·동사 추출 → 공통 테마 파악
   - 예: "홈 추천 배너 개선", "홈 피드 추천 로직 수정" → `홈 추천` 그룹

### 클러스터 이름 규칙

- 3~8자 이내 한국어 테마명
- 패턴: `{영역} {기능}` (예: `홈 추천`, `검색 개인화`, `상품 매칭`)
- 단일 티켓 클러스터도 허용 (억지 묶음 금지)

### 클러스터 내 정렬

각 클러스터 내 티켓은 다음 순서로 정렬:
1. Priority 내림차순 (Highest > High > Medium > Low)
2. Created 오름차순 (오래된 요청 우선)

---

## Step 4: 마크다운 리포트 생성

`output/ticket_review_{YYYYMMDD}.md`에 저장한다.

### 리포트 구조

```markdown

> **분석 파일**: `{CSV 파일명}`
> **전체 티켓**: {N}건 / **MATCH 대상**: {N}건 / **판단 필요**: {N}건 / **제외**: {N}건

---

## 요약

| 구분 | 건수 |
|------|------|
| MATCH 확정 | {N}건 |
| 클러스터 수 | {N}개 |
| 판단 필요 | {N}건 |
| MATCH 제외 | {N}건 |

> **주목 클러스터**: {가장 많은 티켓 보유 클러스터 이름} ({N}건) — {1줄 설명}

---

## MATCH 처리 과제 — 클러스터별

| 티켓 | 제목 | 유형 | 우선순위 | 상태 | 담당자 |
|------|------|------|---------|------|------|
| [TM-123](#) | 제목 | Story | High | Open | 홍길동 |
| ...  |  |  |  |  |  |

**공통 특징**: {레이블, 에픽, 키워드 등 묶인 근거}

## 판단 필요 과제

> 아래 티켓은 MATCH 도메인 여부가 불명확합니다. 직접 확인 후 배정하세요.

| 티켓 | 제목 | 판단 보류 사유 | 검토 포인트 |
|------|------|--------------|------------|
| TM-456 | 제목 | "추천" 언급이나 UX 개선 요청 성격 혼재 | 추천 로직 변경 포함 여부 확인 필요 |

---

## MATCH 제외 과제

<details>
<summary>제외 목록 보기 ({N}건)</summary>

| 티켓 | 제목 | 제외 사유 |
|------|------|----------|
| TM-789 | 제목 | UX/UI 단순 수정, 추천 로직 무관 |

</details>
```

### 작성 원칙

- 클러스터 설명은 PM이 바로 이해할 수 있는 비즈니스 언어 사용
- 티켓 제목은 원문 그대로 (수정 금지)
- 판단 보류 사유는 구체적으로: "키워드 X 있으나 Y 성격도 혼재" 형식
- 제외 목록은 `<details>` 접기 처리 (리포트 가독성 유지)

## Step 5: Self-Validation

| 검증 항목 | 통과 기준 |
|----------|---------|
| CSV 파싱 완료 | 전체 행 수 = 파싱된 티켓 수 (헤더 제외) |
| 분류 완비 | 모든 티켓이 확정/판단필요/제외 중 하나에 속함 |
| 클러스터 존재 | MATCH 확정 티켓 1건 이상이면 클러스터 1개 이상 |
| 리포트 저장 | `output/ticket_review_{YYYYMMDD}.md` 파일 존재 |

실패 시 해당 단계 재실행 (최대 2회).

## 오류 처리

| 오류 | 처리 |
|------|------|
| CSV 파일 없음 | 폴더 경로 안내 후 중단 |
| CSV 파싱 실패 (인코딩 등) | UTF-8 → EUC-KR 순서로 재시도. 실패 시 사용자에게 파일 확인 요청 |
| 필수 컬럼 부재 (`Key`, `Summary`) | 어느 컬럼이 없는지 명시 후 중단 |
| 전체 티켓 MATCH 제외 | "MATCH 대상 과제가 없습니다. CSV 내용을 확인해주세요." 후 중단 |
