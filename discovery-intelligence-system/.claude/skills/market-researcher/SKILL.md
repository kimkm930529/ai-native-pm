# market-researcher — 웹 리서치 및 시장 데이터 수집 규약

## 개요

`market-analyst` 에이전트가 경쟁사 분석과 시장 규모 산정 시 활용하는
웹 리서치 절차, 데이터 출처 우선순위, 가정 명시 규칙을 정의하는 보조 스킬.

---

## 1. 데이터 수집 우선순위

시장 데이터 수집 시 아래 순서로 출처를 활용한다:

| 우선순위 | 출처 유형 | 신뢰도 | 예시 |
|---------|---------|--------|------|
| 1순위 | 정부·공공 통계 | 최고 | 통계청, 산업통상자원부, 과기부 |
| 2순위 | 업계 리서치 리포트 | 높음 | 대한상의, 코트라, 닐슨코리아 |
| 3순위 | 주요 언론 보도 | 중간 | 매일경제, 한국경제, 조선비즈 |
| 4순위 | 글로벌 리서치 | 중간 | Statista, Grand View Research, CB Insights |
| 5순위 | Bottom-up 추정 | 낮음 (명시 필수) | 사용자 수 × ARPU 기반 역산 |

**5순위 사용 시**: 반드시 `[추정값]` 태그를 붙이고 가정 목록에 추가.

## 2. 경쟁사 리서치 절차

### Step 2-A: 경쟁사 식별

WebSearch 또는 WebFetch를 활용하여 아래 키워드로 탐색한다:

```
검색 쿼리 패턴:
- "{탐색 주제} 서비스 비교 2025 2026"
- "{탐색 주제} 앱 추천 한국"
- "{탐색 주제} 경쟁사 시장 분석"
- "site:platum.kr OR site:byline.network {탐색 주제}"
- "{영문 도메인} Korea alternative"
```

### Step 2-B: 경쟁사별 데이터 수집 항목

각 경쟁사에 대해 아래 항목을 수집한다. **확인 불가 항목은 "N/A (미확인)"으로 표기.**

```
- 서비스명 / 운영사
- 주요 기능 목록
- 한국어 / 로컬화 지원 여부
- 타겟 사용자 세그먼트
- 비즈니스 모델 (무료 / 유료 / 수수료 / 광고)
- 가격 정책 (확인 가능한 경우)
- MAU 또는 가입자 수 (공개 자료 기준)
- 주요 수상 / 투자 이력
- 알려진 약점 (사용자 리뷰, 언론 보도 기반)
```

### Step 2-C: 한국 시장 적합도 평가

아래 5개 기준으로 각 경쟁사의 한국 시장 적합도를 평가한다:

| 기준 | 가중치 | 평가 방법 |
|------|--------|----------|
| 한국어 지원 완성도 | 30% | 앱/웹 UI 언어, CS 채널 |
| 결제 수단 다양성 | 20% | 카카오페이, 네이버페이, 신용카드 |
| 로컬 콘텐츠/데이터 | 25% | 한국 특화 데이터 보유 여부 |
| 국내 규제 준수 | 15% | 개인정보보호법, 전자상거래법 |
| 파트너십/제휴 | 10% | 국내 기업과의 협력 관계 |

점수: 1(매우 낮음) ~ 5(매우 높음), 가중 평균으로 최종 점수 산출.

## 3. TAM/SAM/SOM 산정 방법론

### 3-A. Top-down 방식 (1순위~3순위 데이터 있을 때)

```
TAM = 공식 통계의 전체 시장 규모
SAM = TAM × 자사가 실제 서비스 가능한 세그먼트 비율
SOM = SAM × 현실적 점유율 목표 (초기 3~5년 기준)
```

**SOM 점유율 기준:**
- 신규 진입자: 0.5~2%
- 기존 사업 확장: 5~15%
- 강력한 브랜드 기반: 15~30%

### 3-B. Bottom-up 방식 (데이터 부족 시)

```
TAM = 잠재 사용자 수 × 연간 지불 가능 금액(ARPU)
SAM = 실제 온라인/앱 사용 가능 사용자 수 × ARPU
SOM = 현실적 획득 가능 사용자 수 × ARPU
```

**가정 명시 템플릿:**
```
[가정-TAM-01] 한국 {타겟 세그먼트} 인구: {N}만 명
  출처: 통계청 {연도} 인구조사 또는 "추정"
[가정-TAM-02] 온라인 서비스 침투율: {N}%
  출처: {출처} 또는 "유사 카테고리 평균 추정"
[가정-ARPU-01] 연간 1인당 지출: {N}만 원
  출처: {출처} 또는 "경쟁사 가격 정책 기반 추정"
```

### 3-C. Conservative vs Optimistic 시나리오 분기

| 변수 | Conservative | Optimistic |
|------|-------------|-----------|
| 시장 성장률 | 업계 평균 -20% | 업계 평균 +30% |
| 온라인 침투율 | 현재 수준 유지 | +15~20%p |
| 자사 점유율 | 최저 추정치 | 카테고리 리더 추정치 |

## 4. 화이트스페이스 발굴 방법론

### 4-A. 기능 갭 분석

기능 비교 매트릭스에서 "경쟁사 모두 ❌인 행"을 우선 화이트스페이스 후보로 선정.

추가로 아래 패턴을 탐색:
- ⚠️(부분 지원)이 많은 기능 → "불완전한 솔루션" 공백
- 경쟁사가 유료(🔒)로만 제공하는 기능 → "접근성 공백"
- 한국어 미지원 기능 → "로컬화 공백"

### 4-B. 사용자 리뷰 신호 탐색

경쟁사 앱스토어 리뷰 또는 커뮤니티(레딧, 인스타, 카카오·네이버 카페)에서
반복적으로 언급되는 불만을 수집한다:

```
검색 쿼리:
- "{경쟁사명} 단점 OR 불편 OR 아쉬운 site:reddit.com OR site:ppss.kr"
- "{경쟁사명} 리뷰 단점 2025 2026"
- "앱스토어 {경쟁사명} 별점 1~2점 리뷰 분석"
```

## 5. 데이터 신뢰도 경고 기준

수집한 데이터에 아래 상황이 발생하면 보고서에 경고 태그를 붙인다:

| 상황 | 경고 태그 | 처리 방법 |
|------|---------|----------|
| 3년 이상 된 데이터 | `[오래된 데이터 — 검증 필요]` | 더 최신 데이터 탐색 후 없으면 기재 |
| 출처 불명확 | `[출처 미확인]` | 삭제 또는 "추정" 명시 |
| 단일 출처 의존 | `[단일 출처 — 교차 검증 필요]` | 추가 출처 탐색 권장 |
| 글로벌 데이터를 한국에 적용 | `[글로벌→한국 추정]` | 한국 침투율 조정 가정 명시 |

## 6. 리서치 결과 캐시

수집된 원본 데이터는 아래 파일에 저장하여 재분석 시 재활용한다:

```
output/research-cache/
├── competitors_{YYYYMMDD}.json   ← 경쟁사 원본 데이터
├── market-size_{YYYYMMDD}.json   ← 시장 규모 원본 수치
└── reviews_{YYYYMMDD}.json       ← 사용자 리뷰 수집 원본
```

**캐시 유효기간**: 30일. 이후 재수집 권장.
