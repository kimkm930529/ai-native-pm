# Report Agent System — 오케스트레이터 (CLAUDE.md)

## 역할

C레벨 보고를 위한 산출물을 생성하고 검토하는 에이전트 시스템.
PRD·2-Pager 이외의 전략 보고서, 투자 케이스, 로드맵 보고서, 의사결정 문서 등을 대상으로 한다.

---

## 1. 시스템 구조

```
report-agent-system/
├── CLAUDE.md                        ← 이 파일 (오케스트레이터)
├── references/
│   └── report_principles.md         ← 좋은 보고서 원칙 (검토 기준)
├── output/                          ← 검토 결과 저장
└── .claude/agents/
    └── report-red-team/
        └── AGENT.md                 ← C레벨 보고서 품질 검토 에이전트
```

---

## 2. 지원 작업 유형

| 작업 | 설명 | 서브에이전트 |
|------|------|------------|
| **보고서 품질 검토** | 기존 문서를 C레벨 관점에서 검토, 개선 피드백 | `report-red-team` |
| (확장 예정) 보고서 생성 | 인터뷰/메모 기반 C레벨 보고서 초안 작성 | — |
| (확장 예정) 템플릿 적용 | 상황별 보고서 구조 제안 | — |

---

## 3. 보고서 품질 검토 워크플로우

```
사용자 요청 (보고서 파일 경로)
        │
        ▼
오케스트레이터: 파일 확인 + 보고서 유형 식별
        │
        ▼
report-red-team 에이전트 호출
        │
        ▼
output/report_review_{YYYYMMDD}_{주제}.md 저장
        │
        ▼
요약 출력 (질문 수, 카테고리 분포, 상위 Critical 3개)
```

### 보고서 유형 식별 기준

| 유형 | 특징 | 검토 강조점 |
|------|------|-----------|
| 투자 케이스 | ROI 계산, Build vs Buy | 숫자 근거, 반론 처리 |
| 전략 로드맵 | 분기별 실행 계획 | 실행 가능성, 마일스톤 |
| 현황 보고 | 지표·진행 현황 공유 | So What, 의사결정 요청 |
| 제안 보고 | 새로운 투자/방향 제안 | Ask 명확성, 반론 처리 |

---

## 4. report-red-team 호출 규약

```
Task: 아래 경로의 보고서를 C레벨 보고서 품질 기준으로 검토해줘.
파일: {보고서 파일 경로}
보고서 유형: {투자 케이스 / 전략 로드맵 / 현황 보고 / 제안 보고 / 기타}
검토 기준: report-agent-system/references/report_principles.md 참조
산출물: report-agent-system/output/report_review_{YYYYMMDD}_{주제}.md
```

---

## 5. 오류 처리

| 상황 | 처리 |
|------|------|
| 파일 경로 미제공 | 사용자에게 보고서 경로 요청 |
| 보고서 유형 불명확 | 자동으로 "기타"로 처리 후 검토 진행 |
| 보고서가 너무 짧음 (500자 미만) | "보고서가 충분히 완성되지 않았습니다" 안내 후 진행 여부 확인 |
